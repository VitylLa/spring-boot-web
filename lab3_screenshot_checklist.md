# Список скріншотів для лабораторної №3 (у правильному порядку)

## 1) package.json зі scripts
- Етап: налаштування scripts.
- Вікно: редактор коду (файл `frontend/package.json`) або браузерний скрін із згенерованого артефакту.
- Команда в консолі: `cat frontend/package.json`.
- Що має потрапити в кадр: блок `scripts` з `dev`, `build`, `preview`, `lint`.
- Підпис: **Рис. 1. Налаштовані scripts у файлі package.json.**

## 2) .env та .env.production
- Етап: налаштування змінних оточення.
- Вікно: редактор або термінал.
- Команда в консолі: `cat frontend/.env && echo "-----" && cat frontend/.env.production`.
- Що має потрапити в кадр: значення `VITE_APP_STATUS=Development` і `VITE_APP_STATUS=Production Mode`.
- Підпис: **Рис. 2. Файли .env та .env.production зі змінною VITE_APP_STATUS.**

## 3) Інтерфейс у режимі Development
- Етап: перевірка dev-режиму.
- Вікно: браузер.
- Команда в консолі: `npm run dev -- --host 127.0.0.1 --port 5173`.
- Що має потрапити в кадр: сторінка застосунку з `Mode: Development`.
- Підпис: **Рис. 3. Інтерфейс застосунку в режимі Development.**

## 4) Результат npm run lint
- Етап: лінтування.
- Вікно: термінал.
- Команда в консолі: `npm run lint`.
- Що має потрапити в кадр: успішне завершення без помилок.
- Підпис: **Рис. 4. Результат виконання npm run lint (0 помилок).**

## 5) Результат npm run build
- Етап: генерація артефакту збірки.
- Вікно: термінал.
- Команда в консолі: `npm run build`.
- Що має потрапити в кадр: рядки зі згенерованими файлами та розмірами.
- Підпис: **Рис. 5. Результат виконання npm run build.**

## 6) Структура папки dist
- Етап: аналіз артефакту.
- Вікно: термінал.
- Команда в консолі: `find dist -maxdepth 2 -type f | sort`.
- Що має потрапити в кадр: `dist/index.html`, `dist/assets/index-*.js`, `dist/assets/index-*.css`.
- Підпис: **Рис. 6. Структура папки dist після успішної збірки.**

## 7) Інтерфейс у режимі Production Mode
- Етап: перевірка production через preview.
- Вікно: браузер.
- Команда в консолі: `npm run preview -- --host 127.0.0.1 --port 4173`.
- Що має потрапити в кадр: сторінка застосунку з `Mode: Production Mode`.
- Підпис: **Рис. 7. Інтерфейс застосунку в режимі Production Mode.**

- Етап: запуск перегляду production-артефакту.
- Вікно: термінал.
- Команда в консолі: `npm run preview -- --host 127.0.0.1 --port 4173`.

- Підпис: **Рис. 8. Результат запуску npm run preview.**

## 9) git status
- Етап: гігієна репозиторію.
- Вікно: термінал.

- Етап: перевірка ignore-правил.
